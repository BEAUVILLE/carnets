# Carnets DIGIYLYFE

**Carnets DIGIYLYFE** est le dépôt public destiné à porter les contenus textuels de doctrine, de pédagogie terrain et de référencement naturel autour de DIGIYLYFE.

Ce dépôt ne remplace pas la vitrine principale. Il la renforce.

- La vitrine donne l’évidence.
- Le clip donne l’émotion.
- Le QR déclenche l’action.
- Les carnets donnent la mémoire, les mots, la doctrine et le référencement.

## Positionnement

DIGIYLYFE ne vend pas seulement une application.

DIGIYLYFE vend la route entre le client et le professionnel — **sans détour, sans commission, sans dépossession**.

Les carnets servent à expliquer cette route avec des mots simples, humains et enracinés dans le terrain : boutiques, locations, chauffeurs, services, jobs, paiement direct, QR professionnel, visibilité locale et souveraineté numérique.

## Premières pages à produire

### 1. doctrine.html
Page mère. Elle explique la différence DIGIYLYFE : contact direct, paiement direct, 0% commission, respect du professionnel, souveraineté numérique.

### 2. qr-professionnel-senegal.html

Page pédagogique sur le paiement direct au professionnel : Wave, cash, virement, TPE existant ou autre méthode choisie par le terrain.

## Règles éditoriales

Chaque carnet doit respecter ces règles :

1. Écrire pour les humains avant d’écrire pour Google.
2. Parler simple, clair et terrain.
3. Éviter les textes IA froids, génériques ou remplis de jargon.
4. Relier chaque page à la vitrine principale.
5. Relier chaque page aux actions utiles : tarifs, clip, contact, QR, module concerné.
6. Ne jamais promettre ce que le terrain ne peut pas encore tenir.
7. Protéger la doctrine : DIGIYLYFE aide le professionnel, ne le remplace pas.

## Déploiement GitHub Pages

1. Créer un dépôt public, par exemple :

```txt
BEAUVILLE/digiylyfe-carnets
```

2. Déposer les fichiers à la racine du dépôt :

```txt
index.html
README.md
```

3. Activer GitHub Pages :

```txt
Settings → Pages → Deploy from branch → main → /root
```

4. Ajouter le domaine personnalisé :

```txt
carnets.digiylyfe.com
```

5. Dans le DNS du domaine, créer un CNAME :

```txt
carnets → BEAUVILLE.github.io
```

6. Quand GitHub Pages est actif, vérifier que le site ouvre bien :

```txt
https://carnets.digiylyfe.com/
```
